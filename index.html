Experience

Personal Project - Spending Tracker

This Spending Tracker web app is a simple app that lets users enter their expenses, categorize them, and see how much they're spending from month to month or year to year. This is a live application that anyone is welcome to use, available here. It is still very much in beta, and uses free-tier hosting services, so the site can be slow at times. I plan to continuously update the site, and include features such as adding the ability to change currency.

I made this to use personally but also to learn how to host an application online (this is the first full project I've ever hosted). It was made with HTML, CSS, and Vanilla JavaScript for the frontend and an ASP.NET Core backend.

Personal Project - Keepsuite

KeepSuite is a web app for tracking what books you've read and what you thought about them. Users can enter when they read the book, a summary and review of it, as well as select an overall reaction to it from a list of options. Users can also do the same things with movies, television shows, and documentaries.

My girlfriend, Louise Jakobsson, and I made this project together, with a focus on making a very smooth, satisfying, and user-friendly experience (many websites have the opposite experience). For the frontend we opted for Vue and tried out a backend framework called NestJS, a framework built on top of Node.js which uses TypeScript.

This project is not yet live, but we plan to release a beta version very, very soon.

Work Experience - Happy at Work

I was extremely lucky to have the chance to work on a real project during my studies. Happy at Work is a startup whose software service tracks employee feedback anonymously. There I worked primarily on the backend of the service, which was an ASP.NET project.

Beyond that, I learned other critical processes which are extremely important when working on a project in a team, like using Git in an environment where there are several developers, agile processes like scrum and kanban (we tried both while I was there), continous integration, and the flow of using Dev, Test, and Production environments.
